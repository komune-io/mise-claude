# claude-env: Standalone CLI for Declarative Claude Code Environment Management

**Date:** 2026-04-20
**Status:** Approved
**Replaces:** mise-claude Lua backend plugin

## Problem

The current mise backend plugin approach has fundamental issues:

1. **Scope mismatch** ‚Äî mise is a version manager; claude-env does config management (`.mcp.json`, `.claude/settings.json`). The abstraction doesn't fit.
2. **Fragility** ‚Äî npm dependency issues (tinyexec), `pwd` assumptions, errors swallowed by pcall.
3. **Concurrency/locking** ‚Äî mkdir spinners, stale locks, race conditions from mise's parallel execution model.

## Solution

A standalone Rust CLI (`claude-env`) that reads a single config file and orchestrates Claude Code tool setup. Sequential execution eliminates locking. Direct file writes eliminate the scope mismatch. Rust eliminates the fragility.

## Config Format

File: `claude-env.toml` at project root.

```toml
[mcp]
context7 = "2.1.4"
chrome-devtools = "1.0.3"
shadcn = "0.2.1"

[skills]
"vercel-labs/next-skills/next-best-practices" = "latest"
"anthropics/claude-skills/web-design-guidelines" = "latest"

[plugins]
"anthropics/claude-code/code-review@claude-code-plugins" = "latest"
"anthropics/claude-code/feature-dev@claude-code-plugins" = "latest"
"obra/superpowers/superpowers@superpowers" = "latest"

[cli]
get-shit-done-cc = "1.22.4"
bmad-method = "0.5.0"
```

Sections define tool type. No prefix parsing needed.

Short aliases (e.g., `context7` ‚Üí `@upstash/context7-mcp`) resolved via built-in registry. Full npm package names also accepted directly (e.g., `"@someorg/my-custom-mcp" = "1.0.0"`).

## Lockfile

File: `claude-env.lock` at project root. Committed to git.

```toml
# Auto-generated by claude-env. Do not edit.

[mcp.context7]
package = "@upstash/context7-mcp"
version = "2.1.4"
integrity = "sha512-abc123..."

[mcp.chrome-devtools]
package = "chrome-devtools-mcp"
version = "1.0.3"
integrity = "sha512-def456..."

[skills."vercel-labs/next-skills/next-best-practices"]
version = "1.0.0"
resolved_at = "2026-04-20"

[plugins."anthropics/claude-code/code-review@claude-code-plugins"]
version = "latest"
resolved_at = "2026-04-20"

[cli.get-shit-done-cc]
package = "get-shit-done-cc"
version = "1.22.4"
integrity = "sha512-ghi789..."
```

- Pins exact resolved versions even when config says `"latest"`.
- `integrity` (sha512) for npm packages ‚Äî detects supply chain tampering.
- `resolved_at` for non-npm tools (no integrity hashes available).
- `claude-env install` uses lockfile; `claude-env update` refreshes it.

## CLI Commands

```
claude-env install          # Install from lockfile (or resolve + create lockfile if missing)
claude-env update           # Check all tools for updates, show changelogs, update lockfile
claude-env update <tool>    # Update a single tool
claude-env diff <tool>      # Show changelog between pinned and latest version
claude-env list             # Show installed tools, versions, available updates
claude-env add <tool>       # Add tool to config (detects type, queries versions)
claude-env remove <tool>    # Remove from config + clean generated config files
```

## Install Mechanics

Sequential pipeline on `claude-env install`:

1. **Read** `claude-env.toml` + `claude-env.lock`
2. **Resolve** ‚Äî determine what needs doing (fresh install, upgrade, already satisfied)
3. **Install each tool in order:**

| Type | Action | Output |
|------|--------|--------|
| **mcp** | `npm install <pkg>@<ver> --prefix ~/.claude-env/packages/<name>` | Write entry to `.mcp.json` |
| **cli** | Same npm install | Run `post_install`, skip `.mcp.json` |
| **skills** | `npx skills add <owner/repo> --skill <name> -a claude-code -y` | Skills in `.claude/skills/` |
| **plugins** | `claude plugin marketplace add` + `claude plugin install --scope project` | Registered in `.claude/settings.json` |

4. **Write** updated `claude-env.lock`
5. **Report** results

### Key Design Decisions

- **Global package cache at `~/.claude-env/packages/`** ‚Äî no per-project `node_modules`. `.mcp.json` points to absolute paths.
- **Sequential execution** ‚Äî single process, one tool at a time. No locking needed.
- **Idempotent via lockfile + installed state** ‚Äî compares lockfile version against `~/.claude-env/packages/<name>/package.json` (npm) or presence of expected output files (skills/plugins). Skip if satisfied. No marker files.
- **Direct `.mcp.json` writes** ‚Äî read, merge, write. No concurrent writers.
- **`post_install` runs immediately** ‚Äî PATH includes `~/.claude-env/packages/<tool>/node_modules/.bin`. No deferred execution.

## Tool Registry

Built into the binary. Maps aliases to npm packages and defines per-tool overrides.

**Aliases:**
| Alias | Package |
|-------|---------|
| context7 | @upstash/context7-mcp |
| chrome-devtools | chrome-devtools-mcp |
| shadcn | shadcn |
| gsd | get-shit-done-cc |
| bmad | bmad-method |
| openspec | @fission-ai/openspec |

**Overrides (post_install, bin_name, extra_deps):**

| Package | Override |
|---------|----------|
| shadcn | `bin_name = "shadcn"`, `post_install = "shadcn mcp init --client claude"`, `extra_deps = ["tinyexec@1.0.2"]` |
| get-shit-done-cc | `post_install = "get-shit-done-cc --claude --local"` |
| bmad-method | `post_install = "bmad-method install --directory ${PROJECT_ROOT} --modules bmm --tools claude-code --yes"` |
| @fission-ai/openspec | `post_install = "openspec init --tools claude"` |

Unknown tools (not in registry) work directly ‚Äî use the full npm package name in config.

## Error Handling

- **Fail per-tool, not globally** ‚Äî one broken tool doesn't stop others.
- **Show actual errors** ‚Äî no swallowing, full stderr from failed commands.
- **Exit codes** ‚Äî 0 = success, 1 = partial failure, 2 = config error.
- **`--verbose`** ‚Äî shows exact commands being run.
- **Known-issue hints** ‚Äî registry can surface known npm breakage.

```
$ claude-env install

  ‚úì mcp/context7         2.1.4 installed
  ‚úì mcp/chrome-devtools  1.0.3 installed
```

## Distribution

- GitHub releases (prebuilt binaries for macOS arm64/x86_64, Linux x86_64)
- `cargo install claude-env`
- Homebrew tap
- Optionally: `mise install claude-env` (mise is great at installing standalone binaries)

## Non-Goals

- Not a version manager ‚Äî doesn't manage Node, Rust, etc.
- Not a task runner ‚Äî no `claude-env run`.
- No concurrent/parallel installs ‚Äî simplicity over speed.
- No plugin system ‚Äî the registry is built-in. External tools use full package names.
